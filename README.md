# Fix-Traffic-Pipeline
## Games Fix Connections
### Traffic Pipeline Engine – Game Fix Platform

## Описание

**Fix Traffic Pipeline** — платформа для автоматического применения сетевых «фиксов» к онлайн-играм (Apex Legends, Hunt: Showdown, Roblox и др.).

Система состоит из двух частей:

### Python-бэкенд (`data/`)
Выполняет мониторинг сетевых соединений:
- Сканирует активные соединения целевых процессов
- Выполняет обратный DNS-резолвинг (для статусов, указанных в `monitor.yaml`)
- Ведёт списки IP/доменов (используются внешним `winws.exe`)
- Записывает обнаруженные адреса в текстовые файлы (с буферизацией и автоматической записью)
- Интервал сброса буфера (`list_flush_interval`) задаётся в каждом игровом конфиге

### C# WPF-лаунчер (`PlatformLauncher/`)
Графический интерфейс и административная логика:
- Отображает список доступных фиксов (пресетов)
- Скачивает и сохраняет конфигурации игр (YAML)
- Добавляет/удаляет правила брандмауэра Windows (порты TCP/UDP)
- Управляет Cloudflare WARP (установка, запуск, остановка, проверка статуса)
- Создаёт и восстанавливает бэкапы папки `lists`
- Запускает мониторинг (в обычном режиме или только просмотр)
- Синхронизирует список пресетов с GitHub
- Отображает вывод Python-скриптов в терминальном окне
- Применяет темы оформления (7 кастомных YAML-тем)
- Выполняет сброс DNS и сетевых настроек («Починить интернет»)
- Прописывает Cloudflare-домены в списки

**Важно:** вся логика, кроме мониторинга (порты, WARP, бэкапы), вынесена в C#. Python-скрипт `monitor.py` запускается лаунчером с нужными аргументами и только собирает статистику соединений.

## Запуск

1. Запустите `PlatformLauncher.exe` **от имени администратора**
2. Лаунчер автоматически:
   - Проверит наличие Python 3.13 и установит зависимости в виртуальное окружение `.venv` (из WHL-файлов в `extra/python/`)
   - Создаст пустой `presets.yaml`, если он отсутствует
   - Попытается найти папку `lists` (через процесс `winws.exe` или распакует из `extra/zdy.zip`)
   - Загрузит список пресетов из `presets.yaml`
   - Применит настройки темы, шрифта и размера терминала из `app_config.yaml`

## Конфигурация

### Общий конфиг (`data/app_config.yaml`)

Содержит настройки, общие для Python и C#:

### Игровые конфиги (`data/configs/<game_id>.yaml`)

Содержат специфичные для игры настройки:

- `target_processes` – процессы для мониторинга
- `cloudflare_domains`, `include_domains`, `exclude_domains` – доменные списки
- `include_ips`, `exclude_ips`, `pass_ips` – IP-списки
- `ports` – порты TCP/UDP для брандмауэра
- `scan_interval`, `logged_connections_max`, `dns_timeout` – технические параметры
- `list_flush_interval` – интервал сброса буфера (по умолчанию 5.0 сек, в игровых конфигах обычно 2.0)
- `list_rules` – правила добавления в списки по статусам соединений

### Пресет мониторинга (`data/configs/monitor.yaml`)

Специальный конфиг для режима «только просмотр» — без ограничения по процессам и без записи в списки. Содержит настройки DNS-разрешения, фильтрации консоли и технические параметры.

## Основные возможности интерфейса

### Вкладка «Фиксы»

- **Список игр** – отображает доступные фиксы с фильтрацией (только установленные / только локальные)
- **Установка** – скачивает YAML-конфиг игры с GitHub и применяет правила портов (брандмауэр)
- **Удаление** – удаляет конфиг и все правила портов для игры
- **Запуск фикса** – создаёт бэкап `lists`, запускает WARP (если включено в настройках) и стартует мониторинг (требует прав администратора)
- **Мониторинг** – запускает только просмотр соединений (без изменения списков и бэкапов, доступен без прав)
- **Свойства** – настройка WARP для конкретной игры (сохраняется в `user_settings.json`)

### Вкладка «Сервис»

- **WARP**:
  - Запустить WARP – устанавливает (при необходимости) и подключает Cloudflare WARP
  - Отключить WARP – отключает WARP
  - Прописать в листы – добавляет Cloudflare домены из `app_config.yaml` в списки
- **Темы** – выбор темы оформления из 7 загруженных YAML-тем (папка `data/themes`):
  - Cyberpunk, Digital Lavender, Fluent Dark, Fluent Light, Mocha Mousse, Neo Brutalist, Ocean Deep
- **Монитор** – вкладка-заглушка («в разработке»)
- **Помощь**:
  - Починить интернет (сброс DNS) – выполняет сброс DNS-кэша, Winsock, TCP/IP и перенастраивает DNS-серверы на Cloudflare WARP

### Дополнительные функции

- **Кнопка «Обновить пресеты»** – синхронизирует `presets.yaml` с удалённым репозиторием
- **Кнопка «LISTS»** – контекстное меню для выбора или открытия папки `lists`
- **Кнопка «Команды»** – контекстное меню для выполнения Python-команд (удобно для отладки)
- **Трей-иконка** – сворачивание в трей, двойной клик для восстановления окна

## Добавление новой игры

1. Создайте YAML-конфиг в `data/configs/<game_id>.yaml` (см. примеры)
2. Добавьте запись в `presets.yaml` с полями:
   - `id` – уникальный идентификатор
   - `name` – отображаемое имя
   - `config_url` – URL для скачивания конфига (или оставьте пустым для локального)
   - `warp_supported` – поддерживает ли WARP
   - `version` – версия конфига
3. После синхронизации (кнопка «Обновить пресеты») игра появится в списке

## Команды Python (для разработчиков)

`monitor.py` поддерживает следующие аргументы (используются лаунчером):

- `--game <id>` – идентификатор игры
- `--lists-path <path>` – путь к папке lists
- `--monitor-only` – запуск в режиме только просмотра (без записи в списки)
- `--no-filter-processes` – отключить фильтрацию по target_processes

**Пример:**
```bash
python data/monitor.py --game apex --lists-path "C:\path\to\lists"
```

## Логирование

- **Лаунчер** пишет логи в `logs/launcher_<дата>.log` (ротация по дням, до 5 файлов по 1 МБ)
- **Python-монитор** пишет логи в `logs/<game_id>_monitor.log` (ротация до 10 файлов по 100 КБ)
- **Вывод терминала** также сохраняется в `logs/console.log`
- **Отладочный лог** – `debug.log` (подробная трассировка всех событий)

## Примечания для разработчиков

### Права администратора
Обязательны для запуска фиксов (изменение брандмауэра и WARP). Лаунчер перезапускается с `runas`, если прав нет.

### WARP
Для работы требуется установленный `warp-cli`. Лаунчер автоматически скачивает и устанавливает WARP при первом запуске (через `WarpManager.InstallAsync()`). Используется протокол MASQUE.

### Папка lists
Должна существовать и быть доступна для чтения/записи. Обычно она находится рядом с `winws.exe`. Лаунчер пытается найти её автоматически, но пользователь может указать путь вручную через меню «LISTS».

### Синхронизация пресетов
Скачивает только `presets.yaml`. Локальный статус `installed` сохраняется.

### Терминал
Используется `EasyWindowsTerminalControl`, который поддерживает ANSI-цвета и полную эмуляцию консоли. Тема, шрифт и размер настраиваются в `app_config.yaml`.

### Темы оформления
Темы хранятся в `data/themes/` в формате YAML. Каждая тема содержит:
- Цвета фона, текста, акцента
- Настройки для терминала (Light/Dark)
- Цвета элементов управления
- Цвета скроллбара

### Общие настройки
Вынесены в `app_config.yaml`, игровые конфиги содержат только специфичные параметры. Это упрощает добавление новых игр и централизованное изменение внешнего вида и поведения мониторинга.
