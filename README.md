# Fix-Traffic-Pipeline

**Games Fix Connections**

---

## Traffic Pipeline Engine – Game Fix Platform

### Описание

**Traffic Pipeline Engine** — платформа для автоматического применения сетевых «фиксов» к онлайн-играм (Apex Legends, Hunt: Showdown, Roblox и др.).
Система состоит из двух частей:

- **Python-бэкенд** (`data/`) – выполняет **только мониторинг** сетевых соединений:
  - сканирует активные соединения целевых процессов;
  - выполняет обратный DNS-резолвинг (для SYN-пакетов);
  - ведёт списки IP/доменов (используются внешним `winws.exe`);
  - записывает обнаруженные адреса в текстовые файлы (с буферизацией и автоматической записью).
- **C# WPF-лаунчер** (`PlatformLauncher/`) – графический интерфейс и вся административная логика:
  - отображает список доступных фиксов (пресетов);
  - скачивает и сохраняет конфигурации игр (YAML);
  - **добавляет/удаляет правила брандмауэра Windows** (порты TCP/UDP);
  - **управляет Cloudflare WARP** (установка, запуск, остановка, проверка статуса);
  - **создаёт и восстанавливает бэкапы** папки `lists`;
  - запускает мониторинг (в обычном режиме или только просмотр);
  - синхронизирует список пресетов с GitHub;
  - отображает вывод Python-скриптов в терминальном окне.

**Важно:** вся логика, кроме мониторинга (порты, WARP, бэкапы), вынесена в C#.
Python-скрипт `monitor.py` запускается лаунчером с нужными аргументами и только собирает статистику соединений.

---

### Запуск

Запустите `PlatformLauncher.exe` **от имени администратора**.
Лаунчер автоматически:
- проверит наличие Python 3.13+ и установит зависимости в виртуальное окружение `.venv`;
- создаст пустой `presets.yaml`, если он отсутствует;
- попытается найти папку `lists` (через процесс `winws.exe`);
- загрузит список пресетов из `presets.yaml`.

---

### Основные возможности интерфейса

#### Вкладка «Фиксы»
- **Список игр** – отображает доступные фиксы с фильтрацией (только установленные / только локальные).
- **Установка** – скачивает YAML-конфиг игры с GitHub и применяет правила портов (брандмауэр).
- **Удаление** – удаляет конфиг и все правила портов для игры.
- **Запуск фикса** – создаёт бэкап `lists`, запускает WARP (если включено в настройках) и стартует мониторинг (требует прав администратора).
- **Мониторинг** – запускает только просмотр соединений (без изменения списков и бэкапов, доступен без прав).
- **Свойства** – настройка WARP для конкретной игры (сохраняется в `user_settings.json`).

#### Вкладка «Сервис»
- **Запустить WARP** – устанавливает (при необходимости) и подключает Cloudflare WARP.
- **Отключить WARP** – отключает WARP.
- **Темы** – переключение светлой/тёмной темы терминала.

#### Дополнительно
- Кнопка **«Обновить пресеты»** – синхронизирует `presets.yaml` с удалённым репозиторием.
- Кнопка **«LISTS»** – позволяет выбрать или открыть папку `lists`.
- Кнопка **«Ввод»** – контекстное меню для выполнения команд Python (удобно для отладки).

### Добавление новой игры

1. Создайте YAML-конфиг в `data/configs/<game_id>.yaml` (см. примеры).
2. Добавьте запись в `presets.yaml` с полями:
   - `id` – уникальный идентификатор;
   - `name` – отображаемое имя;
   - `config_url` – URL для скачивания конфига (или оставьте пустым для локального);
   - `warp_supported` – поддерживает ли WARP;
   - `version` – версия конфига.
3. После синхронизации (кнопка «Обновить пресеты») игра появится в списке.

### Команды Python (для разработчиков)

`monitor.py` поддерживает следующие аргументы (используются лаунчером):

- `--game <id>` – идентификатор игры.
- `--lists-path <path>` – путь к папке lists.
- `--monitor-only` – запуск в режиме только просмотра (без записи в списки).

Пример:
```bash
python data/monitor.py --game apex --lists-path "C:\path\to\lists"
```

### Логирование

- Лаунчер пишет логи в `logs/launcher.log` (ротация по дням, до 5 файлов по 1 МБ).
- Python-монитор пишет логи в `logs/<game_id>_monitor.log` (ротация до 10 файлов по 100 КБ).
- Вывод терминала также сохраняется в `logs/console.log`.

### Примечания для разработчиков

- **Права администратора** – обязательны для запуска фиксов (изменение брандмауэра и WARP). Лаунчер перезапускается с `runas`, если прав нет.
- **WARP** – для работы требуется установленный `warp-cli`. Лаунчер автоматически скачивает и устанавливает WARP при первом запуске (через `WarpManager.InstallAsync()`).
- **Папка lists** – должна существовать и быть доступна для чтения/записи. Обычно она находится рядом с `winws.exe`. Лаунчер пытается найти её автоматически, но пользователь может указать путь вручную.
- **Синхронизация пресетов** – скачивает только `presets.yaml`. Локальный статус `installed` сохраняется.
- **Терминал** – используется `EasyWindowsTerminalControl`, который поддерживает ANSI-цвета и полную эмуляцию консоли.
