# Traffic Pipeline Engine – Game Fix Platform

## Описание

**Traffic Pipeline Engine** — это платформа для автоматического применения сетевых «фиксов» к онлайн-играм (Apex Legends, Hunt: Showdown, Roblox и другие). Система состоит из двух частей:

- **Python-бэкенд** (`data/`) – выполняет всю бизнес-логику:
  - мониторинг сетевых соединений целевых процессов;
  - обратный DNS‑резолвинг (для SYN‑пакетов);
  - ведение списков IP/доменов (используются внешним `winws.exe`);
  - управление правилами брандмауэра Windows (порты);
  - управление Cloudflare WARP;
  - создание и восстановление бэкапов папки `lists`.

- **C# WPF-лаунчер** (`PlatformLauncher/`) – графический интерфейс пользователя:
  - отображает список доступных фиксов (пресетов);
  - позволяет устанавливать/удалять фиксы;
  - запускает мониторинг (в обычном режиме или только мониторинг без прав администратора);
  - управляет WARP (запуск/остановка с проверкой статуса);
  - синхронизирует список пресетов с GitHub (файл `presets.yaml`);
  - показывает вывод Python-скриптов в своём окне.

**Важно:** вся логика (порты, бэкапы, WARP, мониторинг) **вынесена в Python**. Лаунчер только запускает Python-скрипты с нужными аргументами и отображает их вывод. Это позволяет избежать дублирования кода и упрощает поддержку.

## Запуск

Запустите `PlatformLauncher.exe` (от имени администратора).
Лаунчер автоматически:
- проверит наличие Python и установит зависимости;
- создаст пустой `presets.yaml`, если он отсутствует;
- попытается найти папку `lists` (через процесс `winws.exe`);
- загрузит список пресетов из `presets.yaml`.

### Основные возможности интерфейса

- **Вкладка «Фиксы»** – список игр.
  - **Установка** – скачивает конфиг игры (YAML) и применяет правила портов.
  - **Удаление** – удаляет конфиг и правила портов.
  - **Запуск фикса** – запускает мониторинг с созданием бэкапа `lists` и применением портов (требует прав администратора).
  - **Мониторинг** – запускает только мониторинг соединений без изменения портов и бэкапов (доступен без прав).
  - **Свойства** – настройка WARP для конкретной игры (сохраняется в `user_settings.json`).

- **Вкладка «Сервис»** – управление WARP:
  - кнопки «Запустить WARP» и «Остановить WARP»;
  - статус WARP проверяется автоматически при входе на вкладку.

- **Кнопка «Обновить пресеты»** – синхронизирует локальный `presets.yaml` с удалённым репозиторием (`https://raw.githubusercontent.com/AITISPEC/Fix-Traffic-Pipeline/main/data/configs/presets.yaml`). Все локальные статусы установки сохраняются.

## Добавление новой игры

1. Создайте YAML-конфиг в `data/configs/` с именем `<game_id>.yaml`.
2. Добавьте запись в `presets.yaml` (или дождитесь синхронизации, если файл уже есть на GitHub).
3. Конфиг должен содержать:
   - `target_processes` – список процессов для мониторинга.
   - `ports` – TCP/UDP порты для правил брандмауэра.
   - Дополнительно: `include_domains`, `exclude_domains`, `dns_resolve_statuses` и др.

Формат конфига описан в примерах (`apex.yaml`).

## Команды Python (для разработчиков)

`monitor.py` поддерживает следующие аргументы (используются лаунчером):

- `--install-rules <game_id>` – применить правила портов для игры.
- `--remove-rules <game_id>` – удалить правила портов.
- `--warp-status` – вывести `connected` или `disconnected`.
- `--warp-connect` – запустить WARP.
- `--warp-disconnect` – остановить WARP.
- `--game <id> --lists-path <path>` – запустить полноценный мониторинг (с бэкапами).
- `--monitor-only` – запустить мониторинг без бэкапов и записи в `lists`.

Все команды выводят прогресс в `stdout`, что позволяет лаунчеру отображать его в своём окне.

## Логирование

- Лаунчер пишет логи в `logs/launcher.log` (ротация до 1 МБ, 5 файлов).
- Python-монитор пишет логи в `logs/<game_id>_monitor.log` (ротация до 100 КБ, 10 файлов).
- Ошибки и предупреждения также дублируются в консоль лаунчера.

## Примечания для разработчиков

- **Права администратора** – обязательны для запуска фиксов, так как требуется изменение брандмауэра и работа с WARP. Лаунчер перезапускается с `runas`, если прав нет.
- **WARP** – для работы требуется установленный `warp-cli`. Лаунчер не устанавливает WARP автоматически – это делает Python-скрипт при первом вызове (`WarpManager.ensure_started()`).
- **Папка lists** – должна существовать и быть доступна для чтения/записи. Обычно она находится рядом с `winws.exe`. Лаунчер пытается найти её автоматически, но пользователь может указать путь вручную.
- **Синхронизация пресетов** – скачивает только `presets.yaml` по прямой ссылке. Статус установки (`installed`) сохраняется локально.
- **Добавление новых игр** – достаточно разместить конфиг в `configs/` и обновить `presets.yaml`. Лаунчер подхватит изменения после перезагрузки или нажатия «Обновить пресеты».
