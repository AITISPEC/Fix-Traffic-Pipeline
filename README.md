# 📘 Fix Traffic Pipeline – Руководство пользователя и разработчика

## 1. Введение

**Fix Traffic Pipeline** — десктопное приложение для автоматического обхода блокировок и улучшения сетевого соединения в играх и приложениях с использованием **zapret** и **Cloudflare WARP**. Приложение позволяет управлять сетевыми правилами, мониторить соединения, устанавливать и запускать фиксы для конкретных игр.

---

## 2. Основные возможности

- 🎮 **Управление фиксами для игр** (Apex Legends, Hunt: Showdown, Roblox и др.)
- 📦 **Автоматическая установка и обновление конфигов** с GitHub
- 🔥 **Обход DPI** через zapret (списки IP/доменов)
- 🌐 **Интеграция с Cloudflare WARP** (MASQUE-протокол)
- 📊 **Мониторинг сетевых соединений** в реальном времени (вкладка «Мониторинг»)
- 🖥️ **Встроенный терминал** с подсветкой статусов соединений
- 🎨 **Сменные темы** оформления (темная/светлая)
- ⚙️ **Управление правилами брандмауэра** (TCP/UDP порты)
- 💾 **Автоматическое резервное копирование** и восстановление списков
- 📋 **Логирование** с ротацией (синхронное/асинхронное)

---

## 3. Требования к системе

- **Windows 10/11** (x64)
- **Права администратора** (для работы с брандмауэром и процессами)
- **Интернет-соединение** (для загрузки конфигов и WARP)
- **.NET 8.0 Runtime** (встроен в приложение)
- **Python 3.13** (опционально, используется встроенный при необходимости)

## 4. Установка и настройка

### 4.1 Скачивание и запуск
1. Скачайте последний релиз из репозитория.
2. Распакуйте архив в любую папку.
3. Запустите `FixTrafficPipeline.exe` **от имени администратора**.

### 4.2 Первоначальная настройка

#### Шаг 1: Установка zapret
- Перейдите во вкладку **Сервис → ZDY**.
- Нажмите **«Установить Zapret»**.
  Приложение скачает и распакует архив zapret в папку `zdy/` рядом с исполняемым файлом.
- Укажите путь к папке `lists` (автоматически подставляется, но можно выбрать вручную).

#### Шаг 2: Настройка Python
- Перейдите во вкладку **Сервис → Python**.
- Нажмите **«Создать окружение»** (если системный Python не найден, будет использован встроенный).
- Дождитесь установки зависимостей.

#### Шаг 3: Установка Cloudflare WARP (опционально)
- Перейдите во вкладку **Сервис → WARP**.
- Нажмите **«Установить WARP»** (MSI-установщик будет скачан автоматически).
- После установки можно запускать/останавливать WARP через соответствующие кнопки.

## 5. Использование приложения

### 5.1 Главное окно (вкладка «Фиксы»)

- **Список игр** – отображаются все доступные пресеты.
- **Фильтры и поиск** – поиск по названию, сортировка (по алфавиту/статусу), фильтры (установленные/не установленные/пользовательские).
- **Действия над игрой**:
  - **«Скачать»** – загружает конфигурационный YAML-файл с GitHub.
  - **«Запустить»** – запускает фикс для выбранной игры (требует скачанный конфиг и папку `lists`).
  - **«Мониторинг»** – запускает режим наблюдения за соединениями без записи в списки.
  - **ПКМ по игре** – контекстное меню: установка/удаление портов, свойства.

- **Консоль** – отображает вывод Python-скрипта, логи работы, статус соединений.
  Поддерживает контекстное меню (правый клик): проверка версии Python, PIP, обновление пакетов и т.д.

### 5.2 Вкладка «Сервис»

#### ZDY (zapret)
- Выбор папки `lists` (ручной или автоматический поиск).
- Запуск `service.bat` и `general.bat`.
- Установка/обновление zapret.

#### WARP
- Статус (подключён/отключён).
- Запуск/остановка WARP.
- Прописывание доменов Cloudflare в списки (кнопка «Прописать в листы»).

#### Python
- Проверка наличия системного Python.
- Создание виртуального окружения (если системный Python не найден – распаковывается встроенный).
- Установка зависимостей из `requirements.txt` и локальных `.whl`-файлов.

#### Помощь
- **«Починить интернет»** – сброс DNS-кэша, Winsock, перезапуск адаптеров.
- **«Починить Python»** – открывает страницу с инструкцией по исправлению путей Python.

### 5.3 Вкладка «Настройки»

#### Темы
- Светлые и тёмные темы оформления.
  Выберите тему – она применяется к интерфейсу и терминалу.
  *Во время активной сессии смена тем блокируется.*

#### Монитор
- Настройки мониторинга (в разработке).

#### Дебаг
- Включение/выключение записи подробных логов в `debug.log`.
  При переключении предлагается перезапустить приложение.

## 6. Архитектура проекта

### 6.1 Слои (Clean Architecture)

| Слой | Назначение |
|------|------------|
| **Presentation** | WPF-интерфейс (MVVM, Views, ViewModels, адаптеры терминала и тем) |
| **Core** | Интерфейсы сервисов и UseCases (бизнес-логика) |
| **Domain** | Модели данных (AppConfig, GameConfig, GamePreset, ThemeItem и др.) |
| **Infrastructure** | Реализации сервисов (логгер, управление процессами, работа с сетью, бэкапы и т.д.) |
| **Data** | Python-скрипты мониторинга, конфигурационные YAML-файлы, темы |

### 6.2 Ключевые компоненты

- **DI-контейнер** – Microsoft.Extensions.DependencyInjection, регистрация в `DiSetup.cs`.
- **Логгер** – синхронный (`Infrastructure.Logging.Logger`), пишет в `logs/launcher_*.log`, ошибки дублируются в `debug.log`.
- **DebugLogger** – статический, пишет в `debug.log`; ошибки всегда, остальное – по флагу `debug_enabled`.
- **SessionOrchestrator** – оркестрирует сессию: бэкап, WARP, запуск/остановка Python, восстановление.
- **PythonProcessManager** – управляет процессом `monitor.py` через виртуальное окружение.
- **WarpManager** – управление Cloudflare WARP (установка, запуск, остановка, статус).
- **NetworkMonitor** (Python) – сканирует соединения, резолвит DNS, обновляет файлы списков.

### 6.3 Основные технологии

- **WPF / HandyControl** – интерфейс.
- **EasyWindowsTerminalControl** – встроенный терминал с поддержкой ConPTY.
- **YamlDotNet** – сериализация YAML-конфигов.
- **Newtonsoft.Json** (только для user_settings).
- **psutil / watchdog / pyyaml** – Python-зависимости.

## 7. Устранение неполадок

### 7.1 Ошибка SSL при загрузке с GitHub
- Убедитесь, что в `App.xaml.cs` включены протоколы TLS 1.2/1.3:
  ```csharp
  ServicePointManager.SecurityProtocol = SecurityProtocolType.Tls12 | SecurityProtocolType.Tls13;
  ```
- Проверьте доступ к интернету и работу антивируса/фаервола.

### 7.2 Логгер не пишет
- Проверьте, что папка `logs` существует и имеет права на запись.
- Убедитесь, что в `app_config.yaml` параметр `logging.debug_enabled` установлен в `true` (для подробного лога).
- Используется синхронный логгер – ошибки записи дублируются в `debug.log`.

### 7.3 Python не устанавливается
- Убедитесь, что в папке `extra/python` есть все `.whl`-файлы и `get-pip.py`.
- Если используется системный Python – проверьте его версию (должна быть 3.13).
- При создании venv может потребоваться интернет для скачивания зависимостей.

### 7.4 WARP не подключается
- Проверьте, что служба `Cloudflare WARP` запущена.
- Попробуйте переустановить WARP через кнопку «Установить WARP».
- Вручную запустите `warp-cli status`, чтобы увидеть ошибки.

### 7.5 Фикс не запускается
- Убедитесь, что папка `lists` существует и содержит файлы (`ipset-all.txt`, `list-general.txt` и т.д.).
- Проверьте, что целевой процесс игры запущен (например, `Apex.exe`).
- Посмотрите вывод в терминале – там будут сообщения об ошибках.

## 8. Для разработчиков

### 8.1 Сборка проекта

```bash
git clone https://github.com/AITISPEC/Fix-Traffic-Pipeline.git
cd Fix-Traffic-Pipeline
dotnet restore
dotnet build -c Release
```

### 8.2 Структура папок

```
Fix-Traffic-Pipeline/
├── Core/                 # Интерфейсы и UseCases
├── Domain/               # Модели данных
├── Infrastructure/       # Реализации сервисов
├── Presentation/         # WPF-интерфейс (MVVM)
├── Helpers/              # Утилиты
├── Data/                 # Python-скрипты, конфиги, темы
│   ├── configs/          # YAML-конфиги игр
│   ├── src/              # Исходники Python-скриптов
│   ├── themes/           # YAML-файлы тем
│   └── app_config.yaml   # Глобальная конфигурация
├── extra/                # Сторонние зависимости (python, WARP.msi, zdy.zip)
├── logs/                 # Логи (создаётся автоматически)
└── libs/                 # Библиотеки (загружаются при старте)
```

### 8.3 Конфигурационные файлы

#### `app_config.yaml`
- `app_version` – версия приложения.
- `terminal` – настройки терминала (тема, шрифт, ширины колонок).
- `lists` – имена файлов списков.
- `logging` – уровень лога, размер файла, `debug_enabled`.
- `cloudflare_domains` – список доменов Cloudflare для WARP.

#### `presets.yaml` (data/configs)
Список игр с полями:
- `id` – уникальный идентификатор (например, `apex`).
- `name` – отображаемое имя.
- `config_url` – URL для загрузки YAML-конфига.
- `warp_supported` – поддержка WARP.
- `version` – версия пресета.
- `installed` – установлен ли фикс (обновляется автоматически).

#### Конфиг игры (например, `apex.yaml`)
- `target_processes` – процессы, за которыми следить.
- `include_domains` / `exclude_domains` – домены для списков.
- `include_ips` / `exclude_ips` – IP-адреса.
- `ports` – TCP/UDP порты для брандмауэра.
- `list_rules` – правила для статусов соединений (SYN_SENT, ESTABLISHED и т.д.).

### 8.4 Тестирование

В проекте отсутствуют юнит-тесты (в планах). Ручное тестирование:
- Проверка загрузки конфигов и синхронизации.
- Запуск фикса для каждой игры.
- Переключение тем.
- Создание/удаление venv Python.
- Установка/удаление WARP.

### 8.5 Отладка

- Включите `debug_enabled: true` в `app_config.yaml`.
- Смотрите `debug.log` – в него пишутся все подробности (включая сообщения из Python).
- Для отладки Python-скриптов можно запустить `monitor.py` вручную с параметрами.
